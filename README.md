# تطبيق بهار - إدارة المخزون (PWA)

هذا المجلد جاهز لرفعه إلى GitHub وتشغيله كتطبيق موبايل (PWA) يتثبت من المتصفح مباشرة.

## الملفات
```
bahar-pwa/
├── index.html          ← التطبيق نفسه (الواجهة)
├── manifest.json        ← معلومات التطبيق (الاسم، الأيقونة، الألوان)
├── service-worker.js    ← يخلي التطبيق قابل للتثبيت ويفتح بسرعة
└── icons/                ← أيقونات التطبيق بمقاسات مختلفة
```

## الخطوة 1: ارفع الملفات إلى GitHub

1. روح إلى [github.com](https://github.com) وسوي حساب إذا ماعندك.
2. اضغط **New repository** (مستودع جديد). سمّيه مثلاً `bahar-inventory`.
3. خلّيه **Public**.
4. بعد ما يتسوي، اضغط **uploading an existing file** واسحب كل الملفات والمجلدات الموجودة بهذا المجلد (`index.html`, `manifest.json`, `service-worker.js`, ومجلد `icons` بكل محتواه).
5. اضغط **Commit changes**.

## الخطوة 2: فعّل GitHub Pages

1. بصفحة المستودع، روح لـ **Settings** → **Pages** (بالقائمة الجانبية اليسار).
2. تحت **Build and deployment** اختر **Deploy from a branch**.
3. اختر الفرع **main** والمجلد **/ (root)**، واضغط **Save**.
4. انتظر دقيقة أو دقيقتين، بيظهرلك رابط مثل:
   ```
   https://USERNAME.github.io/bahar-inventory/
   ```

## الخطوة 3: ثبّت التطبيق على الموبايل

### أندرويد (Chrome)
1. افتح الرابط من فوق بمتصفح Chrome على الموبايل.
2. بيطلعلك إشعار **"إضافة بهار إلى الشاشة الرئيسية"** تلقائياً، أو اضغط القائمة (⋮) ثم **Install app**.
3. بيصير عندك أيقونة التطبيق على شاشة الموبايل تماماً متل أي تطبيق عادي.

### آيفون (Safari)
1. افتح الرابط من فوق بمتصفح Safari.
2. اضغط زر المشاركة (المربع وفوقه سهم) ثم **إضافة إلى الشاشة الرئيسية**.
3. بيصير عندك أيقونة التطبيق على الشاشة الرئيسية.

بعد التثبيت، التطبيق يفتح بدون شريط المتصفح، بأيقونته الخاصة، وبسرعة — تماماً متل تطبيق حقيقي.

## (اختياري) سوي ملف APK حقيقي

إذا تريد ملف APK تنزله مباشرة (مو بس "إضافة للشاشة الرئيسية")، بعد ما يصير عندك رابط GitHub Pages:

1. روح إلى [pwabuilder.com](https://www.pwabuilder.com)
2. الصق رابط GitHub Pages مالتك بالخانة واضغط **Start**.
3. اختر **Android** ثم حمّل الحزمة.

## ملاحظة مهمة

أي تحديث تسويه على `index.html` بالمستقبل، لازم ترفعه مرة ثانية لنفس المستودع بـ GitHub (نفس خطوة الرفع بالأعلى)، وبعدها التطبيق المثبت عند الموظفين راح يتحدث تلقائياً بفتح بسيط له.
